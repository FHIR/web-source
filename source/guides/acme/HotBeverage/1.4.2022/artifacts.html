Behavior: Operation Definitions

These are custom operations that can be supported by and/or invoked by systems conforming to this implementation guide

hbbrew

The brew operation sends the BrewedBeverage resource to the Coffee/Tea brewing Device
hbstop

The stop operation sends the Device a direction to stop the brew of the beverage (primarily tea) to check for doneness. Used in combination with the Start operation to make the tea just right
hbwhen

When coffee is poured, and milk is offered, it is necessary for the holder of the recipient of milk to say “when” at the time when sufficient milk has been introduced into the coffee. For this purpose, the “WHEN” operation has been added to FHIR.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide

The Hot Beverage to Brew

The parameters to be given to the device to make the requested Hot Beverage.

  • The Machine to make the beverage (optional)
  • The Beverage to be made (required)
  • The type of Milk to be included (optional)
  • The type of Syrup to be included (optional)
  • The type of Alcohol to be included (optional)
  • The type of Sweetener to be included (optional)
  • The type of Spice to be included (optional)
  • Is Coffee to be decaffeinated (shall be NoDecaf)
The AuditEvent for a successful Brewed Hot Beverage

An AuditEvent profile for when request to brew a Hot Beverage action happens successfully.
